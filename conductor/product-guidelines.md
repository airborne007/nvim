# Product Guidelines - Modern Neovim Configuration

## Language & Documentation Style
- **友好且实用 (Friendly & Action-oriented)**：文档和注释应易于理解，侧重于“如何做”和“为什么这么做”。
- **多用示例**：通过具体的代码片段或操作说明，降低用户的配置难度和学习成本。
- **国际化视野**：保持 README 的双语支持，确保国内外开发者都能无障碍使用。

## UI & Design Principles
- **平衡与协调 (Balanced & Harmonious)**：界面设计应在现代美观与系统性能之间取得平衡。
- **适度辅助**：利用图标和颜色提供有效的视觉反馈（如 Git 状态、LSP 诊断），但避免过度装饰导致视觉疲劳。
- **专注导向**：UI 元素应在需要时出现，在编辑时隐退，将屏幕空间最大限度留给代码。

## Engineering & Evolution
- **稳定性至上 (Stability First)**：所有功能的引入必须以不破坏现有工作流为前提。
- **严格测试**：配置的修改应经过实际场景的验证，确保插件之间无冲突，逻辑无死循环。
- **高性能基准**：在保证稳定性的基础上，持续优化加载逻辑，确保启动时间维持在极速水平。
- **模块化维护**：坚持模块化原则，任何新功能的添加都应遵循现有的目录结构，保持代码的整洁与可维护性。
